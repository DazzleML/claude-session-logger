# Human Test Checklist — User-Config Sweep (v0.3.6+ features dormant until #46 fix)

**Phase**: v0.3.7-pre — post-#46 verification
**GitHub**: Refs #46, validates features from #27/#33/#34/#35/#36/#38/#39/#40
**Commit**: (pending Phase 5)

## Why this checklist exists

Github #46 revealed that `hooks/scripts/log-command.py` was calling the legacy `load_configuration()` instead of `ConfigLoader.load()` since v0.3.6. This means **every user-config-driven feature added since v0.3.6 has been silently dead in the live hook**. Unit tests passed (they called `ConfigLoader.load()` directly), and snapshot tests passed (they never set up a user config). Live behavior never exercised any of it.

With #46 fixed, all those features are NOW reachable for the first time. This checklist sweeps each one to verify it actually works end-to-end with a real Claude Code session — not just in isolated test fixtures.

- Phase 4 (#39) and Phase 5 (#40) committed, with #46 fix included

- Plugin marketplace points at the live filesystem; restart Claude Code to ensure fresh code

## Section A — Sanity check: does the hook load user config at all?

Quickest sanity check before exhaustive sweeps.

1. Write `~/.claude/plugins/settings/session-logger.json`:
   ```json
   { "routing": { "subtype_routing": { "bash": true } } }
   ```

2. Restart Claude Code. Run any Bash command.

3. Inspect `~/.claude/sesslogs/<session>/`:
   - [ ] `.bash-bash_*.log` (or `.shell-bash_*.log` depending on routing) exists — proves user config was loaded and subtype expansion fired

If this fails: the #46 fix didn't make it into the deployed hook. Stop and investigate before continuing.

## Section B — Subtype routing per category (v0.3.3, #31)

Each category has a registered subtype extractor in `cclogger/categorize.py`. Setting `routing.subtype_routing.<category> = true` (or a list of subtypes) should produce per-subtype channel files.

For each category below, set the config, restart, fire a matching tool, verify the subtype file appears:

| Category | Extractor key | Config | Expected file pattern |
|----------|---------------|--------|----------------------|
| `bash` | tool name (lowercased) | `"bash": true` | `.shell-bash_*`, `.shell-powershell_*`, `.shell-grep_*`, `.shell-ls_*`, `.shell-glob_*` |
| `meta` | `tool_input.subagent_type` | `"meta": true` | `.agents-help_*`, `.agents-explore_*`, `.agents-senior-engineer_*`, etc. |
| `mcp` | server name from `mcp__<server>__<tool>` | `"mcp": true` | `.tools-todoai_*`, `.tools-zen_*`, etc. (per MCP server) |
| `skill` | `tool_input.skill` | `"skill": true` | `.tools-obsidian_*`, `.tools-commit_*`, etc. (per skill name) |

- [ ] `subtype_routing.bash`: triggers on Bash/PowerShell/Grep/LS/Glob
- [ ] `subtype_routing.meta`: triggers on Agent tool with `subagent_type`
- [ ] `subtype_routing.mcp`: triggers on MCP tools (e.g., todoai MCP)
- [ ] `subtype_routing.skill`: triggers on Skill tool invocations

## Section C — Per-channel `enabled` override

Default: all shipped channels are enabled except `fileio`. Users can flip the default.

NOTE: Bug #45 means overriding `enabled` on an existing channel requires redundantly specifying `file_prefix`. Until #45 is fixed, use the explicit form:

1. Test **enabling** an opt-in channel:
   ```json
   { "routing": { "channels": { "fileio": { "file_prefix": ".fileio_", "enabled": true } } } }
   ```
   - [ ] After Read/Write/Edit tool call, `.fileio_*.log` materializes with full content

2. Test **disabling** a default channel:
   ```json
   { "routing": { "channels": { "convo": { "file_prefix": ".convo_", "enabled": false } } } }
   ```
   - [ ] After User prompt or AI response, `.convo_*.log` is NOT created
   - [ ] Same content still appears in `.sesslog_*.log`

3. Test **disabling tools channel**:
   ```json
   { "routing": { "channels": { "tools": { "file_prefix": ".tools_", "enabled": false } } } }
   ```
   - [ ] AI tool calls (Read, Bash, etc.) do NOT appear in a `.tools_*.log` file (file should not exist)

## Section D — `suppress_markers` per channel (Phase 4, #39)

Channels can opt out of the session-start / context-compacted marker broadcast without disabling the channel entirely.

```json
{
  "routing": {
    "channels": {
      "convo": {
        "file_prefix": ".convo_",
        "options": { "suppress_markers": true }
      }
    }
  }
}
```

- [ ] After session restart, `.convo_*.log` does NOT contain `═══ SESSION START` markers
- [ ] All other channels DO contain the marker
- [ ] Same applies to compaction markers (`═══ CONTEXT COMPACTED`) after `/compact`

## Section E — Per-channel verbosity (Phase 2+3, #38)

ChannelOptions.verbosity supports: string preset (`"full"`, `"preview"`), hint dict (`{"max_chars": N}`), per-role map (with `_default` keyword + per-role overrides).

Test the user-override path for each shape:

### E.1 — string preset
```json
{
  "routing": {
    "channels": {
      "sesslog": {
        "file_prefix": ".sesslog_",
        "options": { "verbosity": "preview" }
      }
    }
  }
}
```
- [ ] `.sesslog_*.log` entries are truncated to ~20 chars (global default for "preview")

### E.2 — hint dict
```json
{
  "routing": {
    "channels": {
      "tools": {
        "file_prefix": ".tools_",
        "options": { "verbosity": { "max_chars": 50 } }
      }
    }
  }
}
```
- [ ] `.tools_*.log` entries truncated to 50 chars (was 100 by default)

### E.3 — per-role map with `_default`
```json
{
  "routing": {
    "channels": {
      "sesslog": {
        "file_prefix": ".sesslog_",
        "options": {
          "verbosity": {
            "_default": "full",
            "write": { "max_chars": 30 },
            "edit": { "max_chars": 30 }
          }
        }
      }
    }
  }
}
```
- [ ] `.sesslog_*.log` Write/Edit entries are truncated to 30 chars
- [ ] Other entries (Bash, Read, etc.) appear at full length

## Section G — Per-channel `newline_policy` override (Phase 2+3, #38)

Two modes: `ESCAPE` (default — `\n` literal) and `RENDER` (real newlines).

```json
{
  "routing": {
    "channels": {
      "shell": {
        "file_prefix": ".shell_",
        "options": { "newline_policy": "render" }
      }
    }
  }
}
```
- [ ] Multi-line Bash output in `.shell_*.log` shows real newlines instead of `\n`
- [ ] `.sesslog_*.log` still shows `\n` (default ESCAPE)

## Section H — Per-channel `role_labels` override (Phase 2+3, #38)

Channels can rename role labels (e.g., display `EDT` instead of `EDIT`).

```json
{
  "routing": {
    "channels": {
      "tools": {
        "file_prefix": ".tools_",
        "options": { "role_labels": { "edit": "EDT", "write": "WRT" } }
      }
    }
  }
}
```
- [ ] `.tools_*.log` Edit entries show `{EDT: ...}` instead of `{Edit: ...}`
- [ ] `.tools_*.log` Write entries show `{WRT: ...}` instead of `{Write: ...}`
- [ ] Other channels unchanged

## Section I — Category route overrides

Reroute a category to different channels (no Bug #45 since this isn't channel-level).

```json
{
  "routing": {
    "category_routes": {
      "io": ["sesslog"]
    }
  }
}
```
- [ ] After Read/Edit/Write, those entries appear ONLY in `.sesslog_*.log`, not `.shell_*`, `.tools_*`, or `.fileio_*`

## Section J — Tool overrides (per-tool routing)

Override routing for a specific tool by name:

```json
{
  "routing": {
    "tool_overrides": {
      "Grep": ["sesslog"]
    }
  }
}
```
- [ ] Grep entries appear ONLY in `.sesslog_*.log`, not `.shell_*` or `.tools_*`
- [ ] Other bash-category tools (Bash, PowerShell, LS, Glob) still route to `[shell, sesslog, tools]`

## Section K — Custom user-defined channel

Add a brand new channel from scratch. Since this is a NEW channel (not an override), `file_prefix` is required by design — Bug #45 does not apply here.

```json
{
  "routing": {
    "channels": {
      "mcp_log": {
        "file_prefix": ".mcp_",
        "enabled": true,
        "options": { "verbosity": "full" }
      }
    },
    "category_routes": {
      "mcp": ["sesslog", "mcp_log"]
    }
  }
}
```

- [ ] After any MCP tool fires (e.g., a todoai MCP call), `.mcp_*.log` is created with the MCP entry
- [ ] `.mcp_*.log` also gets a SESSION START marker (broadcast covers it)

This proves the v0.4.x epic #44 promise — arbitrary user-defined channels via JSON — is achievable today for new channels.

## What mocks don't cover

- **Real `subagent_type` strings**: tests use `"Explore"` / `"help"` synthetic values; actual values vary by Claude Code release (see `private/claude/notes/research/2026-05-12__claude-code-event-surface.md`).
- **Plugin marketplace cache freshness**: if the cache copy is older than the live filesystem, the hook may run stale code; restart Claude Code after pulling fixes.
- **Config file encoding**: real-world users may save JSON with BOM, CRLF line endings, or non-UTF-8 encoding; this checklist assumes standard UTF-8 without BOM.
- **Per-channel split config layout**: most sections above use single-file `session-logger.json`. The split-config layout (`session-logger/channels/<name>.json`) takes precedence when both exist. Sections C-J should be repeated in split-config form once to verify both paths.
