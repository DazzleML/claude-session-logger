## Section 2 — Run counter authority (multi-run verification)

Start a session, then `/exit` and restart Claude Code in the same project. Each run number should increment in lockstep across channels:

- [ ] Run #1 marker in sesslog matches Run #1 in tools/convo/etc
- [ ] After restart, Run #2 marker appears in ALL marker-enabled channels
- [ ] No channel jumps ahead (e.g., tools Run #3 while sesslog still on Run #2)

## Section 3 — Opt-out (manual config edit)

Edit `~/.claude/session-logger.json` (or split-config equivalent) to set `routing.channels.convo.options.suppress_markers = true`. Restart:

- [ ] `.convo_*.log` does NOT receive a new marker after restart
- [ ] `.sesslog_*.log` and others DO receive a new marker
- [ ] Run counter in sesslog still increments correctly

## Section 4 — Compaction marker broadcast

In an active session, trigger context compaction (`/compact` or auto-trigger). Verify:

- [ ] `═══ CONTEXT COMPACTED` marker appears in all enabled marker-bearing channels
- [ ] Compaction counter (`#N`) matches across all channels
- [ ] Session run counter (separate from compaction counter) does NOT increment

## Section 5 — Subtype-derived channels stay clean (regression check)

Enable subtype routing for bash (`routing.subtype_routing.bash = true`) and run a PowerShell command:

- [ ] `.bash-powershell_*.log` is created (or appended to) with the tool entry
- [ ] `.bash-powershell_*.log` does NOT contain a SESSION START marker

## What mocks don't cover

- **Real disk atomicity**: `atomic_append` under concurrent writers is mock-tested but not stressed in production-like load. Test in a session with heavy parallel tool calls.
- **Path resolution edge cases**: subtype paths and reconciled paths are different code paths. The automated tests use `_get_channel_path()` which exercises the live resolver, but visual verification confirms file naming matches expectations.
- **Read-after-write ordering**: in Section 2, the run counter reads sesslog markers BEFORE writing new ones. If file flushing is delayed (rare), an early read could return stale count. Watch for "Run #1 written twice" symptom.
- **`source` file handling**: the `.source` state file is read-and-unlinked. If the file is locked or read-only, marker source detection silently falls back to "session start" instead of "compaction". Compaction tests should verify the `═══ CONTEXT COMPACTED` text appears (not `═══ SESSION START`).
