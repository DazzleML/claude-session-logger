**Phase**: v0.3.7-pre Phase 5

**Commit**: (pending)

## Why this checklist exists

Phase 5 adds a new user-visible channel (`.agents_*`). 17 unit tests cover layers 1-3 of the data-driven channel model (channel exists, route inclusion/exclusion, subtype extraction, options inheritance). What unit tests can't verify: that **a real sub-agent invocation in a live session lands in the right file**, that **subtype splits materialize correctly with real `subagent_type` values**, and that the **Phase 4 marker broadcast applied to agents shows correctly**.

## Section 1 — Default broadcast (channel materializes on first agent call)

After plugin install + Claude Code restart, **spawn any sub-agent** (e.g., type a request that triggers help, explore, or senior-engineer agent). Then inspect the session's sesslog directory:

- [ ] `.agents_*.log` exists in `~/.claude/sesslogs/<session>/`
- [ ] The file starts with a SESSION START marker (`═══ SESSION START` from Phase 4 broadcast)
- [ ] The Task tool invocation appears as `[[ts]] {Task: <description> }` with subagent_type info
- [ ] The same Task entry appears in `.sesslog_*.log` (kitchen sink still captures it)
- [ ] The Task entry does NOT appear in `.shell_*.log` (meta no longer routes to shell)
- [ ] The Task entry does NOT appear in `.tools_*.log` (meta no longer routes to tools — agents has its own dedicated view)

## Section 2 — Subtype routing (per-agent file split)

Enable subtype routing for the `meta` category. Edit `~/.claude/session-logger.json` (or create it if missing) to include:

```json
{
  "routing": {
    "subtype_routing": {
      "meta": true
    }
  }
}
```

Restart Claude Code. Spawn two different agent types (e.g., a help agent and a senior-engineer agent). Then verify:

- [ ] `.agents-help_*.log` exists (specific to help-agent invocations)
- [ ] `.agents-senior-engineer_*.log` exists (specific to senior-engineer-agent invocations)
- [ ] Each per-agent file contains only the matching `subagent_type` entries
- [ ] The undifferentiated `.agents_*.log` still ALSO contains the entries (subtype expansion is additive — see `_expand_with_subtype_channels`)
- [ ] No spurious `.agents-<weird>_*` files for unknown subtypes

## Section 3 — Channel options inheritance (subtype channel inherits parent)

With subtype routing still enabled from Section 2, the derived files should inherit the parent `agents` channel's `ChannelOptions`:

- [ ] `.agents-help_*.log` entries use `verbosity="full"` (same as parent — no truncation)
- [ ] Default formatter is used (rich-format Task entries)
- [ ] No newline policy weirdness (default `ESCAPE` for `\n` characters)

To verify declare-to-override works: add an explicit override to your config:

```json
{
  "routing": {
    "channels": {
      "agents-help": {
        "file_prefix": ".agents-help_",
        "options": {
          "verbosity": {"max_chars": 200}
        }
      }
    }
  }
}
```

After restart and another help-agent call:

- [ ] `.agents-help_*.log` now truncates entries to 200 chars (the explicit override wins)
- [ ] `.agents-senior-engineer_*.log` still uses full verbosity (inheritance unchanged)

## Section 4 — Disable behavior

Set `routing.channels.agents.enabled = false` in your config:

- [ ] After restart + agent invocation, `.agents_*.log` is NOT created
- [ ] Task entries still appear in `.sesslog_*.log` (sesslog routing unchanged)

## What mocks don't cover

- **Real `subagent_type` strings**: the snapshot fixture uses `"Explore"`; real agents emit `help`, `senior-engineer`, `explore`, `oracle`, `investigate`, `general-purpose`, `gpt-codex`, etc. Filesystem sanitization is unit-tested but the full set of real values isn't.
- **Inheritance under live config merge**: the unit tests construct `Config()` directly; only a live session round-trips through `ConfigLoader.load_configuration()` which reads `~/.claude/session-logger.json` and merges with defaults.
- **Phase 4 marker broadcast under real `_get_channel_path` call**: unit-tested but visual confirmation that the divider line shape renders correctly in your terminal is human-only.
- **Plugin cache freshness**: if the plugin cache wasn't updated, the new agents channel won't appear. Re-run `claude plugin install session-logger@dazzle-claude-plugins` if `.agents_*.log` doesn't materialize on first agent call.
